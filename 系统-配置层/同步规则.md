# 同步规则

> **定位**：第二大脑版本同步的安全边界定义。SA 在执行任何「第二大脑有新的更新」操作时，必须先读取本文件，按规则自动分类，不得跳过。
> **版本**：V1.0
> **维护者**：作者（每次发布新版时更新本文件的分级清单）
> **等级**：S1（系统框架·全量覆盖）——用户下载新版时同步规则自身被覆盖。

---

## 一、安全等级定义

| 等级 | 含义 | 默认行为 |
|:--|:--|:--|
| **S1** | 系统框架·全量覆盖 | 新版文件直接替换旧版文件 |
| **S2** | 混合·区段合并 | 按文件内的 PROTECTED 标记精确合并（保护区内用旧版，其余用新版）。无标记 → 降级·整文件保护 |
| **S3** | 用户数据·默认跳过 | 完全不读、不写、不比对。旧版文件保持不动 |
| **S0** | 新增 | 新版有、旧版没有的文件/文件夹 → 创建（含子文件夹全部内容）。旧版有、新版没有 → 不删除 |

## 五、PROTECTED 标记规范

### 标记类型

| 标记 | 含义 | 位置 |
|:--|:--|:--|
| `<!-- SYNC:PROTECTED:START name="标识" -->` | 保护区段开始 | S2 文件任意位置 |
| `<!-- SYNC:PROTECTED:END name="标识" hash="…" -->` | 保护区段结束。hash 由 SA 自动计算并写入 | 与 START 成对 |
| `<!-- SYNC:TABLE_HEADER_END -->` | 表格框架行结束（以下为用户数据行） | S2 保护区段内的表格 |
| `<!-- SYNC:TABLE_DATA_END -->` | 用户数据行结束（以下继续受 PROTECTED 保护） | 同上 |

### 标记规则

1. START 和 END 必须成对出现，name 必须完全一致（区分大小写）。
2. 标记可嵌套的替代方案：PATTERN 标记不支持嵌套。同一文件中多个保护段使用不同 name。
3. hash 字段由 SA 在每次写入保护区内容后自动计算并追加到 END 标记。
4. TABLE 标记是可选的——仅在"区段内表格需要区分框架行和数据行"时使用。
5. 标记是 HTML 注释，在 Obsidian/VS Code 渲染视图中不可见。
