# 结构化思考

> 深度知识协作六阶段法
> 
> 定位：将个人原始思考转化为深度结构化知识产物的标准化协作流程。
> 面向三层用户：人类使用者、AI执行者、第三方智能体。
>
> **闭合性声明**：六阶段法能处理大多数知识整理需求，但并非所有裂缝都能在一个循环中闭合。无法闭合的裂缝是下一轮循环的起点，而非本方法的失败。
>
> **设计决策声明**：六阶段法标准化了**流程**，但刻意不标准化**输出格式**——不同类型的内容需要不同的呈现方式。输出格式由内容类型决定，规则见输出规格书（附件A）。

---

## 〇、调用前：入口分流（Stage 0）

> **目的**：不是所有输入都从阶段1开始。先评估输入状态，判断从哪个阶段切入、走多深、用什么速览格式。
>
> **Stage 0 包含**：输入状态评估 → 深度选择 → 速览格式选择 → 外部内容精度增强 → AI执行规则

### ⚠️ 每次新内容·协议重载（最高优先级）

> SA 每次接收到新内容时，第一步不是评估，而是重载以下核心约束——无论会话已经持续了多少轮，这几条都必须重新推到工作记忆最前面。

```
□ 阶段门禁：每阶段结束后必须停止，等待用户确认，不得自动推进
□ CK1 检查点：7 条必须逐条展示、逐条确认，未确认 → 不得进入 Stage 1、不得移动文件
□ 命名+存放域：必须先确认再归档，不得先移动文件再问
□ 反幻觉：不确定标注【？】，禁止编撰，禁止跳过来源标注
□ 水印：产出末尾必须追加
□ 确认词：每个全量阶段门禁需要用户输入本次会话确认词
```

### ⚠️ 阶段门禁（最高优先级）

无论从哪个阶段切入，SA 必须逐阶段执行，不得跨越。每完成一个阶段 → 展示产出 → 停止 → 等待用户明确说「继续」「进入下一阶段」「可以了」。SA 不得自行判断"用户可能想继续"而自动推进，不得推断沉默为同意。

六个阶段都必须触碰。深度随内容类型调节（全深度/标准/轻量），但阶段本身不缺。

### 输入状态评估表

| 输入特征           | 从哪个阶段切入                    | 示例            |
| -------------- | -------------------------- | ------------- |
| 原始碎片/散落笔记/未结构化 | **阶段1**（结构化）               | 「帮我整理这个新笔记」   |
| 已有骨架但内容单薄      | **阶段2**（深度化）               | 「这份大纲帮我丰富」    |
| 已有完整内容需要检查     | **阶段3**（自检）或**阶段6**（双视角审视） | 「帮我校验这份文档」    |
| 已有内容需要找冲突      | **阶段4**（横向对比）              | 「看看这和我知识库冲突吗」 |
| 已有多个相关内容需要统一   | **阶段5**（纵向整合）              | 「把这些需求文档整合」   |
| 灵感/随手记/一句话备忘   | **轻量流程**（阶段1简→2简→3简）       | 「这个想法记一下」     |

### 深度选择器

| 内容类型 | 建议深度 | 说明 |
|----------|---------|------|
| 核心框架/理论/原创方法论 | **全六阶段** | 需要最严格的质检 |
| 补充观点/案例/佐证材料 | **阶段1+2+3** | 结构化+深度化+自检 |
| 灵感/随手记/一句话备忘 | **轻量流程** | 见下方轻量流程定义 |
| 外部资料/他人观点摘录 | **追问用户意图**：是归档保存（轻量流程）还是深度整合（全阶段）？ | 不自动判断 |
| 个人日记/情绪记录 | **不适用本流程** | 结构化会破坏其价值 |

### 速览格式选择器（与内容类型同步确定）

> **跨会话一致性规则**：同一个内容类型 → 同一种速览格式。不依赖 AI 自行判断。

| 内容类型 | 速览格式 | 结构 |
|----------|---------|------|
| 原创方法论/认知框架 | **概念地图式** | 核心概念A → 衍生B → 实操C |
| 商业框架/分析工具 | **问句式** | 问1→答1章，问2→答2章… |
| 学习方法论/综合指南 | **表格式导航** | 你想做什么 → 去哪里 |
| 工具/流程/操作手册 | **步骤式** | 1→2→3→做 |
| 诊断/排障 | **诊断式** | 如果（症状）→ 则（去X章） |
| 案例/复盘 | **一句话+三数字** | 核心主张 + 关键指标 |

> 具体格式模板参见 [[#附件A：输出规格书]] —— A·速览入口·操作指引

**区分规则**：
- 问句式 vs 诊断式：如果内容有「问题→解法」结构 + 「症状」概念 → 诊断式；如果内容是「领域→问题→答案」结构 → 问句式
- 概念地图式 vs 表格式导航：内容以概念推导为主 → 概念地图式；以场景引导为主 → 表格式导航

### 外部内容精度增强

```
面对外部资料/他人观点摘录，在阶段0追加追问：

「你对这份内容的原作者/原书有多熟悉？
 A. 非常熟悉（可以纠正 AI 的推断）
 B. 大致了解（能判断明显偏差，但细节不确定）
 C. 不太了解（仅靠这份笔记，无法验证 AI 的推断）」

如果答案为 C → 标记「低验证能力场景」→ 
  ① 阶段2：L2 推断增加【替代解读】
  ② 联网搜索：B类缺失补充搜索优先级从「建议做」升级为「优先做」，
     AI 主动声明并申请执行
```

### AI执行规则

```
1. 评估用户输入 → 判断内容类型 + 同步确定速览格式 + **检测是否已有结构化骨架**：
   检测：用户内容是否已有思维导图/大纲/目录层级/结构化标题？
   ├── 有 → SA：「你的内容已经有结构了。你希望：
   │        A. 保留你的结构，我补充缺失的细节和关联
   │        B. 按标准格式重新梳理，可能和你的结构不同」
   │    选 A → 保留用户结构，SA 只做：①核心概念提取 + ②概念边界确认（不超过3个问题）+ ③标注【？】
│            → 展示 → 等待确认 → 确认后进入 Stage 2
│    选 B → 正常走 Stage 1 搭骨架
│
│  **选A后的兼容处理**：六阶段走完后，SA 在文件末尾追加「标准框架映射表」：
│  | 你的结构节点 | 对应的标准概念 | 关系类型 |
│  |:--|:--|:--|
│  | [用户术语] | [SA提炼的标准概念] | 同构/包含/互补 |
│  此映射表使 SA 在做跨文件对比时能将用户术语翻译为标准概念，无需改写用户结构。
   │    选 B → 正常走 Stage 1 搭骨架
   └── 无 → 正常走 Stage 1
2. 告知用户：「建议从阶段X开始，走Y深度，速览格式用Z。是否同意？」
3. **节奏偏好**：告知用户接下来会经历的具体阶段路线图（例：Stages 1→2→3→6），每阶段结束后暂停等待，并主动询问：
   「你想一口气走完这 X 个阶段，还是做完一个阶段先看看再决定？」
   → 用户选一口气 → SA 每阶段结束仍暂停等「继续」，但不额外解释路线图
   → 用户选分批 → SA 每阶段结束额外提醒「下一阶段是 [Stage N]，准备好了说继续」
4. 外部资料 → 追问用户意图（归档还是整合）+ 追问对原内容的了解程度
5. 用户确认后 → 进入对应阶段
6. 如果无法判断内容类型 → 主动问用户
```

### ⚠️ CK1·Stage 0 检查点（强制执行）

> SA 必须在进入 Stage 1 之前在输出中展示此表。**逐条展示、逐条确认**，任何一项未确认 → 不得推进、不得移动文件。

```
[ ] 内容类型：___（核心框架 / 补充 / 灵感 / 外部资料 / 日记）
[ ] 结构处理：___（保留原结构 / 标准格式重建 / 旧链接清理重建 / 不适用）
[ ] 外部资料·处理意图（如适用）：___（归档保存 / 深度整合 / 不适用）
[ ] 外部资料·了解程度（如适用）：___（非常熟悉 / 大致了解 / 不太了解 / 不适用）
[ ] 加工方案：Stage ___ 起 | ___（全六阶段/标准/轻量）| 速览：___ | 节奏：___（一口气/分批）
[ ] 命名建议：___（域前缀·核心概念·框架类型@版本号）
[ ] 存放域：___（01~06）
[ ] 用户确认：以上全部逐条确认后 SA 方可进入加工流程
```

## 一、全流程前置约束

> **约束决定输出质量的下限。以下约束在任何阶段都生效。**

### 核心约束

| # | 约束 | 为什么 | 违反的表现 |
|---|------|--------|-----------|
| C1 | **实事求是，不允许编撰** | 切断「推测→断言」的AI惯性 | 把「可能」说成「显然」、把推断写成已知事实 |
| C2 | **不清楚就说不清楚** | 切断「模糊糊弄」的AI惯性 | 用「一般来说」「某种意义上」逃避不确定性 |
| C3 | **最终确定前不写入文件** | 切断「快点完成」的AI惯性 | 在用户明确说「导出」前创建/覆盖文件 |
| C4 | **推测必须标注** | 切断「脑子里推断的，嘴里说成确定的」 | 不给标注就给结论 |
| C5 | **每个方案必须给理由** | 切断「随便给个方案等反馈」的AI惯性 | 只说「建议改成X」，不说为什么 |
| C6 | **不下沉责任，不转移问题** | 切断「免责声明式方案」 | 加注「以上可能有偏差」而不改内容本身 |

### C3补充：精确区分「展示」和「输出」

| 操作 | 定义 | 是否允许（未经确认前） |
|------|------|----------------------|
| **展示** | 在对话中呈现内容供审阅 | ✅ 允许 |
| **输出** | 创建新文件或覆盖已有文件 | ❌ 禁止 |
| **原地修改** | 修改已有文件的内容 | ❌ 禁止（视为输出） |

> 可以「展示」N个版本，不能「输出」1个字节，直到用户明确说「可导出」。

## 二、六个阶段详解

---

### 阶段1：结构化

**目标**：把散落思考搭成骨架，不确定的地方标注【？】。

#### ⚠️ 分块重读（进入本阶段第一步）

> SA 进入本阶段时必须先重读本阶段的全部内容（操作清单+产出+禁止事项），不得依赖初次通读的记忆。

#### 操作清单

```
1. 读取原文（完整读取，不截断）
2. 识别核心概念（哪些是作者原创的？哪些是引用的？）
3. 搭骨架（概念之间的层级、因果、并列关系）
4. 每个不确定的地方标注【？】
5. 提确认问题（最多5个，只问方向性问题，不纠结措辞）
6. 等待用户确认后再进入下一阶段
```

#### 【？】标注的四种类型

| 类型 | 示例 | 含义 |
|------|------|------|
| 概念边界不明 | 【？此概念是否包含XX？】 | 我不确定这个概念的范围 |
| 关系不明 | 【？A和B是因果关系还是并列？】 | 我不确定两个概念怎么关联 |
| 原文歧义 | 【？此处原文可解读为X或Y】 | 原文有歧义，需作者意图 |
| 缺失感知 | 【？是否缺少Z维度？】 | 我感觉缺了什么但不确定 |

#### 产出

- 结构化骨架（层级大纲/思维导图）
- 【？】标注清单
- 待确认问题（最多5个）

#### ⚠️ CK2·阶段门禁（强制执行）

展示产出 → **停止** → 等待用户输入本次会话确认词 → 用户输入正确确认词后 → SA 更新 `个人配置层/会话状态.md` 的「上次检查点」字段为：`Stage 1·结构化·已完成`。下次会话启动时用于断点恢复。

> 📌 确认后 → SA 输出：「📌 下一步：进入下一阶段。准备好后输入确认词。」

#### 禁止事项

- ❌ 编撰概念（原文没提的不能加）
- ❌ 跳过不确定点
- ❌ 在此阶段展开细节
- ❌ 一次提超过5个确认问题

### 阶段2：深度化

**目标**：逐条展开血肉，每一句话都能回溯到原文。

#### AI知识边界声明

```
如果AI对源内容领域的知识储备有限：
  → 主动声明知识边界
  → L2/L3推断比例会上升
  → 建议用户对关键推断加强确认

如果阶段0标记为「低验证能力场景」：
  → 每个L2推断增加【替代解读】——「也可能是C，取决于原文上下文」
  → 告知用户替代解读的存在，让用户知道这里不止一种理解方式
```

#### ⚠️ 分块重读（进入本阶段第一步）

> SA 进入本阶段时必须先重读本阶段的全部内容（操作清单+产出+禁止事项），不得依赖初次通读的记忆。

#### 操作清单

```
1. 逐条展开骨架的每个节点
2. 每个展开点必须对应原文中某一句话
3. 三级推断标注：
   ├── L1·显式推断：原文说「A→B」，展开成「A→B的路径是C」，C是自然推导
   │   → 标注【推断L1】，可批量展示，不需逐条确认
   ├── L2·扩展推断：原文只说了A，基于A的逻辑推了B，但B可能与作者原意有偏差
   │   → 标注【推断L2】，必须逐条确认
   │   → 低验证能力场景下，追加【替代解读】
   └── L3·跨域推断：引入原文没提的外部概念来解释原文
       → 标注【补充/跨域】，必须逐条确认

4. L2/L3确认时机：每阶段结束时，集中列出本阶段所有L2+补充项，
   一次性提交用户确认。不每发现一个就中断流程。

5. L2自我质疑：每个L2推断提交前，先自问：
   □ 这个推断是否依赖于我对该书/该作者的特殊了解？
   □ 如果是 → 标注「此推断基于训练数据中对该领域的有限认知，优先确认」
   □ 如果不是 → 正常标注

6. 数字编号语义域前缀规则：当产出物中出现多个数字编号时，
   必须为每个编号加语义域前缀。格式：前缀·数字+描述。
   前缀 = 该数字所属的最小语义域，用一个名词概括。
   例：「三级术语递进」→「术语·三级递进」，「四级检验」→「检验·四级标准」
   目的：前缀为AI向量空间提供语义锚点，防止不同数字编号在跨章节引用时混淆。

7. 联网验证关键事实假设（上限20次，详见联网规则）
8. 展开完成后，整体回看：有没有偏离原意？
```

#### 联网规则

```
上限：20次
策略：每次换不同域名的可靠源，允许多路径搜索

提前止损：连续10次搜索（覆盖≥3个不同域名类型）全部失败
  → 主动告知：「联网不可用。以下XX处信息来自训练数据，
     标注为【训练数据推断】，建议基于对原内容的了解进行验证。」
  → 不等20次用完

离线声明：全部失败 → 声明「联网不可用」→ 切离线
离线：依赖训练数据 + 标注「此信息来自训练数据，建议后续人工验证」

联网目的分层：
  A类·事实验证（如：马斯洛×收入相关性） → 正常计数
  B类·缺失补充（如：四轮驱动缺失单元格） → 
      主动声明「这是缺失补充型搜索，建议不限次数。是否同意？」
      → 等用户确认后执行

外部内容增强：低验证能力场景下，B类搜索优先级从「建议做」升级为「优先做」
  → AI主动声明并申请执行，不等用户主动提出
```

#### 产出

- 带三级标注的展开版
- 联网验证记录
- 阶段2末尾：L2+补充项清单（批量提交确认）

#### ⚠️ CK2·阶段门禁（强制执行）

展示产出 → **停止** → 等待用户输入本次会话确认词 → 用户输入正确确认词后 → SA 更新 个人配置层/会话状态.md 的「上次检查点」字段为：$stage。下次会话启动时用于断点恢复。

> 📌 确认后 → SA 输出：「📌 下一步：进入下一阶段。准备好后输入确认词。」

#### 禁止事项

- ❌ 添加原文完全没有的观点且不标注
- ❌ L2/L3写成无标注（把推测伪装成事实）
- ❌ 偏离骨架主方向

### 阶段3：自检

**目标**：强制从生成模式切换到质检模式。逐条审视。

#### ⚠️ 分块重读（进入本阶段第一步）

> SA 进入本阶段时必须先重读本阶段的全部内容（操作清单+产出+禁止事项），不得依赖初次通读的记忆。

#### 操作清单

```
1. 强制分类：每个子节点归入以下四类之一
   ┌─ A类：无问题
   ├─ B类：措辞优化（分B1/B2两级）
   │   ├── B1·明显歧义：不改会误解 → 必须修，不计入上限
   │   └── B2·可优化表述：改了好，不改也能懂 → 计入上限（最多5处）
   ├─ C类：逻辑优化（推理漏洞、因果关系错误）
   └─ D类：结构性修改（概念重叠、分类缺失、层次错位）

2. 重点检查三个维度：
   ├── 重叠检查：同一个意思是否出现在两个不同的名字下？
   ├── 模糊检查：这个节点，一个陌生人读了能理解吗？
   └── 缺失检查：有没有应该出现但没出现的东西？
       → 如果发现可能的缺失，用ABC选择式提问：
         「X没有出现。可能是：
          A. 你认为不重要；B. 打算后续单独展开；
          C. 我理解有偏差，其实它被包含在Y里。你是哪种？」
       → 不要直接问「你是不是忘了写X」

3. C类D类优先于B类——不在措辞上反复折腾
4. 外化关键判断点：列出以下判断点，请用户逐条确认
   ├── 概念的定义是否正确？
   ├── 概念之间的层级/并列/因果是否正确？
   └── 有没有被遗漏的重要概念？
```

#### 自检陷阱防护

| 陷阱 | 表现 | 防护 |
|------|------|------|
| 过度自检 | 在措辞上反复修改，忽略逻辑问题 | 强制四分类 + B2上限5处 |
| 回避真正问题 | 只挑安全的小问题 | C类D类优先于B类 |
| 自查天花板 | AI发现不了自己不理解的东西 | 外化关键判断点 → 请用户确认 |

- 关键判断确认清单

#### 操作清单

```
1. 搜索知识库：关键词匹配 → 找到所有潜在相关文件

3. 逐条比对：

   ├── 框架的步骤是否重叠/冲突？

```
「发现一个张力，需要你判断：

  A说法（来源X）：[原文表述]
  B说法（来源Y）：[原文表述]

  — A可能在说[维度一]

  你的判断：A和B是什么关系？」
```

#### 无冲突时：转为一致性验证

```
如果横向对比未发现P0/P1冲突：

## 三、阶段间机制

### 定向回跳机制

```
> ⚠️ 回跳后必须重新走对应阶段的 CK2 阶段门禁——展示产出 → 停止 → 等待用户确认 → 再推进。不得因为"这是修正，用户应该知道"而跳过门禁。

不是所有回跳都需要推倒重来：

L1·措辞回跳：阶段6发现措辞问题
  → 只回跳阶段2改表述，阶段3-5不重做
  → 阶段2 重新走门禁：展示修正 → 停止 → 等待确认

L2·概念回跳：阶段6发现某个子概念的定义有问题
  → 回跳阶段1修正定义 → 重新走2→3→5（每阶段走门禁）
  → 阶段4横向对比可不重做（除非新定义改变了与已有文件的关系）

L3·框架回跳：阶段6发现核心框架结构问题
  → 回跳阶段1，全流程重走（每阶段走门禁）

触发条件：任何阶段发现前序阶段的输出有偏差
```

### 确认级联防护

```
阶段1确认的概念定义 → 阶段2展开时再次外化（「基于阶段1确认的X定义…」）
阶段5整合时 → 回看阶段1的原始确认记录
阶段6审视时 → 检查当前输出与阶段1确认是否一致

目的：防止微小偏差在六阶段中被逐级放大
```

## 五、完整约束清单

### 第一层：全流程通用

| # | 约束 | 违反判据 |
|---|------|---------|
| C1 | 实事求是，不允许编撰 | 原文没有的断言且无标注 |
| C2 | 不清楚就说不知道 | 用模糊表述回避不确定性 |
| C3 | 最终确定前不输出文件 | 用户说「导出」前写入了文件 |
| C4 | 推测必须标注 | 【推断L1/L2】【补充/跨域】标注缺失 |
| C5 | 方案必须给理由 | 修改建议无「为什么」 |
| C6 | 不下沉责任，不转移问题 | 出现免责声明式表述 |

### 第二层：防幻觉细化规则

| 规则 | 适用阶段 | 说明 |
|------|---------|------|
| 原文没有的数据不能编 | 全阶段 | 收入数据、增长率、年份 → 必须核实或标注为推测 |
| 不能把相关性说成因果性 | 阶段2/5 | 「随收入增长而出现」≠「收入增长导致」 |
| 不能把个例说成规律 | 阶段2/6 | 「有一个案例是X」≠「通常都是X」 |
| 反面案例必须可查证 | 阶段4/6 | 失败案例必须标注来源 |
| 不能把框架「解释力」说成「预测力」 | 阶段5/6 | 「框架能解释成功」≠「按框架必成功」 |

### 第三层：AI知识盲区声明

```
当面对AI知识储备明显不足的领域（如高度专业的医学、法律、尖端科研）：
  → 阶段3的C类D类问题发现力下降
  → 阶段4的概念比对精度下降
  → 阶段6中AI视角的有效性下降
  → 处理：主动声明「此领域超出我的知识储备，
      以下分析仅供参考，建议由领域专家复核」
```

## 六、适用范围与不适用范围

### 适用 ✅

- 原创方法论/认知框架/思维模型
- 知识体系中的核心概念整理
- 多个同类内容的整合统一
- 个人知识库的归档结构化

### 不适用 ❌

- 个人日记/情绪记录（结构化破坏价值）
- 纯数据/纯事实罗列（不需六阶段分析）
- 外部转载/他人观点的纯摘录（不涉及个人知识整合）
- 已过严谨编辑的出版物（除非需与个人知识库对齐）

> **精度声明**：结构化思考对个人原创内容的处理精度高于外部内容。外部内容的 L2 推断修正率因「可纠正性」差异而偏高——这不完全是方法论的局限，也是 AI 能力在外部内容领域的天然边界。三层精度增强（Stage 0 了解程度追问 + Stage 2 替代解读 + 联网优先）可缩小但不能消除此差距。

## 七、AI执行总则

### 默认行为

```
1. 收到新内容 → Stage 0评估 → 告知建议的起始阶段、深度、速览格式
2. 用户确认 → 进入对应阶段
3. 每阶段结束 → 展示产出 → 停止。⚠️ 在用户明确说「继续」「下一阶段」「可以了」之前，SA 不得进行下一阶段的任何操作。SA 不得推断沉默为同意，不得自动推进。
4. 发现前序偏差 → 触发定向回跳（L1/L2/L3）
5. 灵感/随手记 → 走轻量流程 → 有价值则标记建议升级
6. 输出后可选操作（仅在用户明确指令时执行）：
   □ 是否删除原始文件？
   □ 是否覆盖原始文件（原地替换内容）？
   □ 是否需要更新知识库中其他文件的链接？
7. 输出前必检：用附件A·输出规格书扫一遍产出物——必含模块A-F是否齐全？
```

### 关键心态

```
1. 你不是在「完成一个任务」，你是在「和用户共建一个知识产物」
2. 用户是作者，你是编辑——你有建议权，没有最终决定权
3. 每次展示是为了被挑战，不是为了被鼓掌
4. 越深的质疑，越高的质量
```

## 八、总结：六阶段法速查

| 阶段 | 核心动作 | 最主要的禁止事项 |
|------|---------|-----------------|
| Stage 0 | 评估输入→判断起始阶段+深度+速览格式。外部资料追问用户意图+了解程度 | 跳过评估直接进阶段1 |
| 1·结构化 | 搭骨架+标注【？】+提确认问题 | 编撰概念 |
| 2·深度化 | L1/L2/L3三级展开+联网验证（上限20次）。L2/L3在阶段末批量确认。L2提交前自我质疑。数字编号加语义域前缀 | L2/L3不标注 |
| 3·自检 | 四分类+重叠/模糊/缺失检查 | B2超5处、C类D类不报 |
| 4·横向对比 | 搜索知识库→逐条比对→互补检查→P0/P1/P2 | 只看文件名不读内容 |
| 5·纵向整合 | 裂缝整合→不创造新概念。启动前检查外部引用精度 | 创造新概念、掩盖裂缝 |
| 6·双视角审视 | 人类镜头×AI镜头→P0/P1/P2。必含模块A-F齐全检查 | 跳过AI视角 |
| 轻量流程 | 1简→2简→3简→停 | 对随手记走全流程 |

## 附件A：输出规格书

> **定位**：六阶段法产出物的质量标准。不规定格式，但规定「必须包含什么」。
> **产出物不需要维护版本日志**。版本日志是方法论自身的维护工具，不适用于方法论的产出物。

### 必含模块

| 模块           | 判断标准                                                         | 操作指引                                                           |
| ------------ | ------------------------------------------------------------ | -------------------------------------------------------------- |
| **A·速览入口**   | 一个完全不了解本文档的人，能否在 1 分钟内找到他需要的入口？                              | 扫描全文→提取最可能被问到的 3-7 个场景→为每个场景写一行入口→格式按内容类型适配（见 Stage 0·速览格式选择器） |
| **B·核心概念定义** | 核心术语在首次出现时是否有不含歧义的定义？                                        | 术语首次出现→紧跟一句话定义                                                 |
| **C·主体内容**   | 结构由内容决定，无固定格式                                                | —                                                              |
| **D·框架边界声明** | 是否说明了「什么情况下本框架不适用」？                                          | 至少列出 3 种不适用/部分适用的情况                                            |
| **E·知识库关联**  | 是否列出了与本文档相关的知识库其他文件？每个关联附带一句话定义。如适用，同时链接 Skill 版和 Obsidian 版 | 格式：「[[文件名]]——一句话说明它是什么」                                        |
| **F·来源标注**   | 是否标注了内容来源（原创/外部/混合）和所用方法（六阶段法）？末尾是否强制包含水印？                           | 「来源：XXX。打磨：结构化思考·六阶段法」+ 水印（见根配置 §十五）                 |

### 可选模块

| 模块 | 何时保留 |
|------|---------|
| **G·案例** | 当有助于理解框架时 |
| **H·操作清单/检查清单** | 当框架包含可执行步骤时 |

> ⚠️ 产出物**不需要**保留「版本信息」或「版本日志」——那是方法论自身的维护工具。如果内容有明显迭代（如追问链 v1→v2），通过内容本身的标注说明差异，不单独建版本日志块。
