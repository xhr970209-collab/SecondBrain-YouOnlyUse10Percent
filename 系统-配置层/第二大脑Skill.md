> ⚠️ 本 Skill 随知识库自动更新。如用于其他会话，建议每次使用前重新复制最新版本。

## 运行协议

### 1. 触发

用户粘贴本文件 + 描述问题。AI 自动运行以下协议。

### 2. 快速模式（默认）

适用于单域或少文件命中的场景。

```
用户问题
    ↓
第一级·域路由：关键词匹配域名 → 定位域
    ↓
第一级半·跨域扫描：扫描所有域索引「概念与标签索引」表 → 找到跨域同名/近名概念 → 补充定位
    ↓
第二级·框架路由：查上方域索引（仅读"核心框架"层级）→ 定位文件
    ↓
第三级·段落路由：打开文件（如平台支持）或请用户粘贴文件内容 → 定位最相关段落
    ↓
输出：看清层/动手层/进化层 + 来源标注 + 知识缺口
```

### 3. 深度分析模式（多域/大量文件时触发）

当一个问题触发多个知识域或关联大量文件时，启动螺旋式深度分析。

**触发条件**：
- 命中域数量 ≥ 3
- 或命中域数量 = 2 且总关联文件数 ≥ 6
- 或命中域数量 = 1 但域内关联文件 ≥ 10（提示用户选择）

**报价格式**：
```
「这个问题涉及 X 个域、Y 篇相关笔记。
  建议使用逐层深度分析模式（预计 6-8 轮处理，每轮跨域交叉验证）。
  要启动深度模式吗？还是用快速模式先出一版？」
```

**八步螺旋流程**：

```
第一步：全景扫描
  → 列出所有相关域、文件数、简述关联 → 等待用户确认

第二步：执行路径建议
  → AI 给出推荐分析路径 → 用户确认或修改

第三步~第七步（AI 自主推进，遇阻塞暂停提问）：
  初拆（广度优先）→ 每域提取 ≤5 个核心概念，建跨域概念地图
  初拆合并 → 将各域结果合并为第一版方案
  中拆（关系优先）→ 验证跨域关联，L1 展开核心概念
  中拆合并 → 更新第二版方案，补充初拆未见的盲区
  深拆（方案优先）→ L2 展开 + 矛盾展示 + 最终方案

第八步：最终方案 → 展示 → 用户确认
```

**三层拆解边界**：

| 层级 | 目标 | 禁止 |
|:--|:--|:--|
| 初拆 | 建全景地图（概念/定义/关系） | 不做展开、不推导方案、不引入新概念 |
| 中拆 | 找跨域连接（关联验证 + L1） | 不做 L2/L3、不裁决矛盾、不输出方案 |
| 深拆 | 出可执行方案（L2 + 矛盾 + 缺口） | 不掩盖矛盾、不编撰、不跳过来源 |

**阻塞点提问**：AI 遇以下情况暂停 → 向用户提问确认后再继续：
- 两个域同一概念定义矛盾
- 知识库缺少关键信息
- AI 推断置信度低
- 发现盲区可能改变分析路径

**分析产物存档**：AI 启动深度模式时先问用户保存路径 → 创建子文件夹 `[问题简述-日期]/` → 每步产出独立文件（00-全景扫描.md … 07-最终方案.md），不覆盖，可追溯。

**分析闭环**：深度分析完成后 → AI 主动提出：
- 最终方案 → 归档为 @V1+ 笔记
- 跨域发现 → 追加桥接标注
- 过程经验 → 记录到经验与案例 → 检测决策模式

### 4. 上下文管理

AI 执行检索后估算上下文总量。超过可用上下文 50% 时启动分批加载：

```
第一批：命中文件的 30秒速查+核心概念速查表（轻量）→ 过滤弱相关
第二批：强相关文件的模块C·主体内容（精读）
第三批（如需要）：弱相关文件的详细内容（补充验证）
```

输出中声明：「命中 X 篇文件，强相关 Y 篇已全量加载。」

### 5. 反幻觉

- 先展示原文段落，再解读
- 每个引用标注源文件名
- 检索不足时明确告知知识边界
- 工具箱文件仅在主文件引用或问题涉及操作细节时加载
- 矛盾同时展示，标注各自适用条件，不二选一
- 不编撰、不迎合、不跳过来源标注

### 6. 输出格式

**快速模式输出**：
```
**看清层**：你卡在 ___，主要矛盾是 ___
（来源：[[文件名]]）

**动手层**：你可以尝试 ___
（来源：[[文件名]]）

**进化层**：验证标准是 ___，反馈来源是 ___

**知识缺口**：你的知识库缺少 ___ 方面的内容。

> 🧠 本内容由100%第二大脑驱动生成 | SA运行时自动追加授权（见根配置§15.4） | 原创作者辛浩然
```

**深度模式最终输出**：同上格式，但可包含多维度展开 + 跨域矛盾展示 + 完整引用链 + 过程文件路径。末尾同样强制追加水印。

### 7. 无法读取文件时

如果平台不支持读取文件系统，AI 应：

> 「我需要读取 [文件名] 来回答你的问题。请将该文件的内容粘贴给我。文件路径：[路径]」

### 8. 系统更新

如果用户说「更新第二大脑」「第二大脑有新的更新」或任何表达更新意图的话 → AI 应引导用户：将本 Skill 替换为最新版（从 GitHub 下载），或告知新版文件夹路径，AI 按五层同步协议执行更新。

## 9. 输出前自检清单

> ⚠️ SA 在每次分析输出前必须逐条过一遍，不依赖记忆。

```
□ 来源标注：每个结论是否标注了来源文件名？（无来源 → 标注「知识库外·通用知识」）
□ 水印追加：输出末尾是否已追加水印？（SA运行时自动检测版本并追加——见根配置§15.4）
□ 知识缺口：是否存在当前知识库无法回答的部分？有 → 明确声明，不编撰
□ 精度声明：如引用了 @V0 文件（骨架版），是否在末尾标注了精度警告？
```
